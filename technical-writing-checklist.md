# 테크니컬 라이팅 체크리스트

## 📝 구조와 흐름

### 전체 구조
- [ ] 서론-본론-결론 구조가 명확한가?
- [ ] 각 섹션이 논리적으로 연결되는가?
- [ ] 독자가 흐름을 따라가기 쉬운가?
- [ ] 불필요한 내용이나 중복이 없는가?

### 서론
- [ ] 문제 상황이 구체적으로 제시되었는가?
- [ ] 독자의 공감을 얻을 수 있는가?
- [ ] 왜 이 문제를 해결해야 하는지 동기가 명확한가?
- [ ] 글의 목적과 범위가 명시되었는가?

### 본론
- [ ] 문제 해결 과정이 단계별로 설명되는가?
- [ ] 각 단계의 목적과 이유가 설명되는가?
- [ ] 시도한 방법과 실패한 이유가 구체적인가?
- [ ] 최종 해결책에 도달한 과정이 논리적인가?

### 결론
- [ ] 핵심 내용이 요약되었는가?
- [ ] 배운 점이나 인사이트가 명확한가?
- [ ] 향후 개선 방향이 제시되었는가?

---

## 💡 내용의 명확성

### 기술적 정확성
- [ ] 기술 용어가 정확하게 사용되었는가?
- [ ] 코드 예시가 올바르게 작동하는가?
- [ ] 기술적 설명에 오류가 없는가?
- [ ] 버전 정보가 필요한 경우 명시되었는가?

### 설명의 구체성
- [ ] 추상적인 설명보다 구체적인 예시가 있는가?
- [ ] "이렇게", "저렇게" 같은 모호한 표현이 없는가?
- [ ] 숫자나 데이터로 뒷받침되는가?
- [ ] 구체적인 상황과 맥락이 제공되는가?

### 전문 용어 처리
- [ ] 처음 등장하는 전문 용어는 설명되었는가?
- [ ] 약어는 풀어서 설명했는가?
- [ ] 대상 독자에 맞는 용어를 사용하는가?

---

## 🎯 독자 경험

### 가독성
- [ ] 문단 길이가 적절한가? (3-5문장)
- [ ] 문장 길이가 적절한가? (30자 이내 권장)
- [ ] 한 문단에 하나의 아이디어만 담았는가?
- [ ] 띄어쓰기와 맞춤법이 올바른가?

### 시각적 요소
- [ ] 코드 블록이 적절히 사용되었는가?
- [ ] 코드에 주석이나 설명이 필요한 곳에 있는가?
- [ ] 다이어그램이나 이미지가 필요한 곳에 있는가?
- [ ] 표나 리스트로 정리할 수 있는 정보가 있는가?

### 네비게이션
- [ ] 제목 계층 구조가 명확한가? (H1 > H2 > H3)
- [ ] 제목만 읽어도 내용을 파악할 수 있는가?
- [ ] 중요한 부분을 강조했는가? (볼드, 인용 등)
- [ ] 목차나 섹션 링크가 필요한가?

- [ ] 왜 이렇게 구현했는지 이유가 설명되는가?

## 📌 사용 팁

1. **초고 작성 후**: 전체 구조와 흐름 위주로 체크
2. **1차 수정 후**: 내용의 명확성과 기술적 정확성 체크
3. **2차 수정 후**: 독자 경험과 스타일 체크
4. **최종 점검**: 출판 전 체크리스트 확인

모든 항목을 완벽하게 체크할 필요는 없습니다. 글의 목적과 상황에 맞게 선택적으로 사용하세요.
