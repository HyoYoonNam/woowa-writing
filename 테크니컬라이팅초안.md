# React PWA에서 FCM 기반 웹 푸시 알림 구현하기

## 개요

본 문서는 React PWA(Progressive Web App) 환경에서 FCM(Firebase Cloud Messaging)을 활용한 웹 푸시 알림 시스템 구현 과정을 다룬다. 웹 푸시 알림 구현 시 고려해야 할 기술적 요구사항과 실제 구현 방법, 그리고 발생할 수 있는 문제점과 해결 방안을 제시한다.

## 구현 목표

- 웹 브라우저에서 푸시 알림 수신 기능 구현
- 포그라운드(탭 활성 상태)와 백그라운드(탭 비활성 상태) 모두 지원
- 사용자가 알림 ON/OFF를 제어할 수 있는 토글 기능 제공
- PWA 환경에서의 푸시 알림 동작 보장

## 웹 푸시 알림 시스템 아키텍처

![FCM 알림 전체 흐름](https://img1.daumcdn.net/thumb/R1280x0/?scode=mtistory2&fname=https%3A%2F%2Fblog.kakaocdn.net%2Fdna%2FcN2L28%2FbtsQIwULu8i%2FAAAAAAAAAAAAAAAAAAAAAF9jER1bJ1u8nkaurxDKLPzB-WygHs5ftBaUx6urhl4u%2Fimg.png%3Fcredential%3DyqXZFxpELC7KVnFOS48ylbz2pIh7yKj8%26expires%3D1761922799%26allow_ip%3D%26allow_referer%3D%26signature%3DBBHOQ%252FRvx61nNiwvjUidUI5DaGQ%253D)

FCM 기반 웹 푸시 알림 시스템은 다음과 같은 단계로 구성된다:

### 1. 사용자 알림 권한 요청

브라우저의 Notification API를 통해 사용자로부터 알림 권한을 획득한다.

- `Notification.requestPermission()` 메서드 실행
- 사용자 승인이 있어야 이후 단계 진행 가능
- 브라우저 상단에 알림 허용 팝업이 표시됨

### 2. 서비스 워커 등록

백그라운드에서 푸시 메시지를 처리할 서비스 워커를 등록한다.

- `firebase-messaging-sw.js` 파일을 루트 디렉토리에 배치
- `navigator.serviceWorker.register()` 메서드로 등록
- 페이지가 비활성 상태에서도 푸시 알림을 수신할 수 있게 함

### 3. FCM 토큰 발급

Firebase SDK를 통해 브라우저별 고유 식별 토큰을 생성한다.

- `getToken()` 메서드 호출 시 VAPID 키 전달
- 브라우저/디바이스 식별용 FCM Registration Token 생성

### 4. 토큰 서버 저장

발급받은 토큰을 백엔드 서버에 저장하여 사용자와 매핑한다.

- 데이터베이스에 사용자 ID와 토큰 정보 저장
- 이후 특정 사용자에게 푸시 메시지 전송 시 사용

### 5. 서버에서 알림 발송

백엔드 서버에서 Firebase Admin SDK를 통해 메시지를 전송한다.

- Firebase Admin SDK 또는 HTTP v1 API 활용
- 대상 토큰(또는 토큰 배열)에 메시지 전송

### 6. 브라우저에서 알림 수신 및 표시

클라이언트의 서비스 워커가 메시지를 수신하여 알림을 표시한다.

- 서비스 워커에서 푸시 이벤트 수신
- `self.registration.showNotification()` 메서드로 알림 표시

### 2. VAPID 키 생성

VAPID(Voluntary Application Server Identification) 키는 웹 푸시 메시지의 발신자를 식별하는 공개키이다. 이 키는 브라우저가 해당 웹사이트가 푸시 메시지를 보낼 권한이 있는지 확인하는 데 사용되며, FCM 서버에 애플리케이션 신원을 증명하는 역할을 한다.

프론트엔드에서 FCM 토큰을 생성할 때 다음과 같이 사용한다:

```typescript
import { getToken } from "firebase/messaging";
import { messaging } from "./firebase-config";

const token = await getToken(messaging, {
  vapidKey: "BL8x9x9x9x...", // Firebase Console에서 생성한 VAPID 키
});
```

#### VAPID 키 생성 과정

1. Firebase Console의 Cloud Messaging 탭으로 이동
2. "웹 푸시 인증서" 섹션에서 "새 키 쌍 생성" 클릭
3. 생성된 공개키를 복사하여 프로젝트에 적용

#### 서비스 워커 통합

기존 PWA 서비스 워커에 FCM 기능을 추가한다. `public/service-worker.js` 파일을 다음과 같이 구성한다:

```javascript
// 기존 PWA 기능 유지
self.addEventListener("fetch", (event) => {
  event.respondWith(fetch(event.request));
});

// Firebase FCM 기능 추가
try {
  importScripts(
    "https://www.gstatic.com/firebasejs/9.23.0/firebase-app-compat.js"
  );
  importScripts(
    "https://www.gstatic.com/firebasejs/9.23.0/firebase-messaging-compat.js"
  );

  firebase.initializeApp({
    /* Firebase 설정 */
  });
  // FCM 관련 로직 추가
} catch (error) {
  console.warn("Firebase 초기화 실패:", error);
}

// 개발환경에서 MSW(Mock Service Worker) 지원
if (self.location.hostname === "localhost") {
  importScripts("/mockServiceWorker.js");
}
```

## 알림 설정 시스템 설계

알림 토글 기능의 구현 방법을 설명하기 전에, 전체적인 시스템 설계와 사용자 플로우를 다이어그램을 통해 설명한다. 이후 각 플로우별로 실제 구현 코드를 제시한다.

### 초기 로드 시 알림 설정 초기화

사용자가 웹 애플리케이션에 접속할 때 다음과 같은 순서로 알림 설정을 초기화한다:

1. **이전 알림 설정 상태 확인**: 사용자가 이전에 알림을 활성화했는지 확인하고, 활성화된 경우 저장된 FCM 토큰을 로컬 스토리지에서 복원
2. **서비스 워커 상태 확인**: 등록된 서비스 워커가 있는지 확인하고, 없으면 새로 등록
3. **서버 상태 동기화**: 백엔드 API를 통해 실제 사용자의 알림 설정값을 조회하여 로컬 상태와 동기화
4. **UI 초기화**: 조회된 알림 설정값에 따라 토글 스위치 상태를 초기화
5. **포그라운드 메시지 구독**: 탭이 활성 상태일 때도 알림을 받을 수 있도록 `onMessage` 이벤트 리스너 등록

이러한 초기화 과정을 통해 사용자가 토글을 조작할 때마다 알림이 적절히 동작하도록 보장한다.

### 알림 토글 활성화 플로우

사용자가 알림 토글을 ON으로 설정할 때 다음과 같은 과정을 거친다:

1. **알림 권한 요청**: 브라우저의 Notification API를 통해 사용자로부터 알림 권한을 요청
2. **권한 상태 확인**: 권한이 허용되지 않은 경우 사용자에게 권한 허용 안내 페이지를 표시
3. **FCM 토큰 처리**:
   - 로컬 스토리지에 기존 토큰이 있는 경우 재사용
   - 토큰이 없는 경우 새로 발급
4. **서버 동기화**: 발급받은 토큰을 백엔드 API를 통해 서버에 전송하여 알림 설정을 활성화
5. **로컬 상태 업데이트**: 로컬 스토리지에 현재 알림 상태를 저장
6. **UI 업데이트**: 토글 스위치 상태를 활성화로 변경

### 알림 토글 비활성화 플로우

사용자가 알림 토글을 OFF로 설정할 때의 과정은 다음과 같다:

1. **로컬 상태 변경**: 로컬 스토리지의 알림 상태를 비활성화로 업데이트
2. **서버 동기화**: 백엔드 API를 호출하여 서버의 알림 설정을 비활성화 상태로 변경

이 과정을 통해 해당 사용자는 더 이상 푸시 알림을 받지 않게 된다.

### 푸시 알림 수신 처리

실제 푸시 알림이 도착했을 때의 처리 과정은 다음과 같다:

1. **백엔드 메시지 전송**: 새로운 피드백이 등록될 때마다 백엔드에서 해당 사용자에게 푸시 메시지를 전송
2. **클라이언트 상태별 처리**:
   - **포그라운드 상태**: `onMessage` 이벤트 핸들러가 메인 스레드에서 실행
   - **백그라운드 상태**: `onBackgroundMessage` 이벤트 핸들러가 서비스 워커에서 실행
3. **알림 표시**: 각 상태에 따라 적절한 방식으로 사용자에게 알림을 표시

## 핵심 구현 코드

### 서비스 워커 구현

`public/service-worker.js` 파일에 FCM 기능을 통합한 서비스 워커 구현:

```javascript
// public/service-worker.js
// 기존 PWA 기능 유지
self.addEventListener("fetch", (event) => {
  event.respondWith(fetch(event.request));
});

// FCM 기능 추가
try {
  importScripts(
    "https://www.gstatic.com/firebasejs/9.23.0/firebase-app-compat.js"
  );
  importScripts(
    "https://www.gstatic.com/firebasejs/9.23.0/firebase-messaging-compat.js"
  );

  firebase.initializeApp({
    // Firebase 프로젝트 설정
  });

  const messaging = firebase.messaging();

  // 백그라운드 메시지 처리
  messaging.onBackgroundMessage((payload) => {
    const title = payload.notification?.title || "새 알림";
    const options = {
      body: payload.notification?.body || "",
      icon: "/logo192.png",
      requireInteraction: true,
    };
    self.registration.showNotification(title, options);
  });
} catch (error) {
  console.warn("Firebase 초기화 실패:", error);
}

// 개발환경에서 MSW(Mock Service Worker) 지원
if (self.location.hostname === "localhost") {
  importScripts("/mockServiceWorker.js");
}
```

### NotificationService 클래스

알림 시스템의 핵심인 `NotificationService` 클래스는 토큰 생성, 권한 관리, 서버 통신 등 모든 알림 관련 기능을 담당한다:

```typescript
// services/notificationService.ts
export class NotificationService {
  static async enable(): Promise<NotificationServiceResult> {
    try {
      // 1. 브라우저 지원 여부 확인
      const supportCheck = isNotificationSupported();
      if (!supportCheck) {
        throw new Error("이 브라우저는 푸시 알림을 지원하지 않습니다.");
      }

      // 2. Firebase 설정 확인
      if (!messaging || !VAPID_KEY) {
        throw new Error("Firebase 설정이 올바르지 않습니다.");
      }

      // 3. 알림 권한 요청
      const permission = await this.requestPermission();
      if (permission !== "granted") {
        throw new Error("알림 권한이 거부되었습니다.");
      }

      // 4. 서비스 워커 등록 확인
      const registration = await this.ensureServiceWorkerRegistration();

      // 5. 기존 토큰 확인 (중복 생성 방지)
      const existingToken = getStoredFCMToken();
      let token = existingToken;

      if (!existingToken) {
        // 6. FCM 토큰 생성
        token = await getToken(messaging, {
          vapidKey: VAPID_KEY,
          serviceWorkerRegistration: registration,
        });

        if (!token) {
          throw new Error("FCM 토큰 생성에 실패했습니다.");
        }

        // 7. 백엔드에 토큰 전송
        await postFCMToken(token);
        setStoredFCMToken(token);
      }

      // 8. 로컬 상태 저장
      setStoredNotificationState(true);

      return {
        success: true,
        message: "알림이 성공적으로 활성화되었습니다.",
        data: { token },
      };
    } catch (error) {
      setStoredNotificationState(false);
      throw new Error(createNotificationErrorMessage(error));
    }
  }

  static disable(): NotificationServiceResult {
    setStoredNotificationState(false);
    return {
      success: true,
      message: "알림이 비활성화되었습니다.",
    };
  }

  // 권한 요청 로직
  private static async requestPermission(): Promise<NotificationPermission> {
    if (!("Notification" in window)) {
      throw new Error("이 브라우저는 알림을 지원하지 않습니다.");
    }

    const currentPermission = Notification.permission;
    if (currentPermission === "granted") {
      return currentPermission;
    }

    return await Notification.requestPermission();
  }

  // 서비스 워커 등록 확인
  private static async ensureServiceWorkerRegistration(): Promise<ServiceWorkerRegistration> {
    if (!("serviceWorker" in navigator)) {
      throw new Error("이 브라우저는 Service Worker를 지원하지 않습니다.");
    }

    const existingRegistration =
      await navigator.serviceWorker.getRegistration();
    if (existingRegistration) {
      return existingRegistration;
    }

    const registration =
      await navigator.serviceWorker.register("/service-worker.js");
    await navigator.serviceWorker.ready;
    return registration;
  }
}
```

### React 훅을 통한 상태 관리

FCM 관련 상태 관리를 위해 다음과 같은 커스텀 훅들을 구현했다:

```typescript
// hooks/useFCMManager.ts
export const useFCMManager = () => {
  const [permission, setPermission] =
    useState<NotificationPermission>("default");
  const [isEnabled, setIsEnabled] = useState(() =>
    getStoredNotificationState()
  );

  // 포그라운드 메시지 처리
  useEffect(() => {
    if (!messaging) return;

    const unsubscribe = onMessage(messaging, (payload: MessagePayload) => {
      if (Notification.permission === "granted") {
        new Notification(payload.notification?.title || "새 알림", {
          body: payload.notification?.body || "새로운 메시지가 있습니다.",
          icon: "/logo192.png",
          data: payload.data,
        });
      }
    });

    return () => unsubscribe();
  }, []);

  const updateState = (enabled: boolean) => {
    setIsEnabled(enabled);
    setStoredNotificationState(enabled);
  };

  return {
    isEnabled,
    permission,
    updateState,
  };
};
```

### 알림 설정 토글 구현

사용자가 직관적으로 알림을 제어할 수 있도록 토글 기능을 구현했다:

```typescript
// hooks/useNotificationSettingsPage.ts
export const useNotificationSettingsPage = () => {
  const { isEnabled: localEnabled, updateState } = useFCMManager();

  // 서버 설정과 로컬 설정 동기화
  const { data: serverSettings } = useQuery({
    queryKey: QUERY_KEYS.notificationSettings(),
    queryFn: getNotificationSettings,
  });

  const updateMutation = useNotificationSettingMutation({
    localEnabled,
    updateState,
  });

  const updateNotificationSetting = (enabled: boolean) => {
    if (enabled === isToggleEnabled || updateMutation.isPending) {
      return;
    }

    // 250ms 딜레이로 UI 반응성 개선
    setTimeout(() => {
      updateMutation.mutate({ enabled });
    }, 250);
  };

  return {
    isToggleEnabled: serverSettings?.data?.alertsOn ?? localEnabled,
    isLoading: updateMutation.isPending,
    updateNotificationSetting,
  };
};
```

## 문제 해결 및 트러블슈팅

### 1. 토큰 중복 생성 문제

**문제**: 사용자가 알림 토글을 반복적으로 조작할 때마다 FCM 토큰이 새로 생성되는 문제가 발생했다.

**원인**: 토큰 재사용 로직이 없어서 매번 새로운 토큰을 발급받고 있었다.

**해결 방법**: 로컬 스토리지에 토큰을 저장하고, 기존 토큰이 있는 경우 재사용하도록 로직을 개선했다.

```typescript
// 기존 토큰 확인 후 재사용
const existingToken = getStoredFCMToken();
let token = existingToken;

if (!existingToken) {
  token = await getToken(messaging, {
    /* ... */
  });
  setStoredFCMToken(token);
}
```

### 2. 권한 상태 동기화 문제

**문제**: 사용자가 브라우저 설정에서 알림 권한을 수동으로 변경했을 때, 애플리케이션 내 상태와 동기화되지 않는 문제가 있었다.

**원인**: 권한 상태 변경을 실시간으로 감지하는 메커니즘이 없었다.

**해결 방법**: 주기적으로 권한 상태를 체크하는 로직을 추가하여 상태 동기화를 보장했다.

```typescript
useEffect(() => {
  const interval = setInterval(() => {
    const updatedPermission = Notification.permission;
    if (updatedPermission !== permission) {
      setPermission(updatedPermission);
    }
  }, 1000);

  return () => clearInterval(interval);
}, [permission]);
```

### 3. iOS Safari PWA 알림 정책 문제

**문제**: iOS Safari에서 PWA 환경에서 알림 권한 요청이 제대로 작동하지 않는 문제가 있었다.

**원인**: iOS Safari는 사용자 상호작용 없이 알림 권한을 요청하는 것을 허용하지 않는 정책이 있다.

**해결 방법**: 페이지 로드 시 즉시 권한을 요청하는 대신, 사용자가 로그인 버튼을 클릭한 후 권한을 요청하도록 UX를 개선했다.
